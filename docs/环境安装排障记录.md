## 9. 为什么前端回退代码后仍然会崩溃

这次实际遇到过一个很典型的现象：

- 前端代码已经回退
- 页面仍然返回 `500`
- 单纯重启前端服务也没有恢复

这类问题不是“代码没回退干净”，而是**前端运行环境已经坏了**。

如果坏的是源码，回退代码通常就够了；如果坏的是进程状态，重启通常就够了。  
但如果坏的是前端依赖或构建产物，例如：

- `frontend/node_modules/next` 不存在
- `frontend/node_modules/next/dist/bin/next` 缺失
- `.next` 构建缓存损坏

那么重启只是把一个坏环境重新启动一次，问题不会消失。

它更像 Python 项目里的这种情况：

- 不是 `.py` 文件写错了
- 而是 `.venv` 里的包目录被删了
- 这时候重启 `uvicorn` 没有意义
- 必须重新同步依赖或重建环境

### 可重复执行的恢复命令

先检查 `next` 是否真实存在：

```bash
cd /Users/admin/Downloads/Code/lumenx
ls frontend/node_modules/next/dist/bin
```

如果这里报 `No such file or directory`，说明前端依赖已经坏了，直接执行下面这组恢复命令：

```bash
cd /Users/admin/Downloads/Code/lumenx/frontend
npm ci
rm -rf .next
npm run build
npm run dev -- --hostname 127.0.0.1 --port 3000
```

如果 `3000` 端口上还挂着旧进程，先查再停：

```bash
lsof -nP -iTCP:3000 -sTCP:LISTEN
kill <PID>
```

然后重新启动前端：

```bash
cd /Users/admin/Downloads/Code/lumenx/frontend
npm run dev -- --hostname 127.0.0.1 --port 3000
```

恢复后建议立即验证：

```bash
curl -I -s http://127.0.0.1:3000
curl -s -o /tmp/lumenx_home.out -w "%{http_code}\n" http://127.0.0.1:3000/
```

如果返回 `200`，说明前端已经恢复。

## 10. 常见问题与排障

### 1. 前端打开后提示 Internal Server Error

常见原因是前端依赖安装不完整，尤其是 `next` 包缺文件，例如：

- `next/dist/compiled/cookie`
- `next/dist/bin/next`

处理方式：

```bash
cd /Users/admin/Downloads/Code/lumenx/frontend
npm ci
rm -rf .next
npm run dev -- --hostname 127.0.0.1 --port 3000
```

### 2. 前端页面能打开，但样式不对

本次已将字体改为本地打包资源，不再依赖 Google Fonts。若仍有样式异常，优先执行：

```bash
cd /Users/admin/Downloads/Code/lumenx/frontend
rm -rf .next
npm run dev -- --hostname 127.0.0.1 --port 3000
```

如果之前依赖装坏过，再补一次：

```bash
npm ci
```

### 3. 环境配置弹窗一直存在

先确认 `[.env](/Users/admin/Downloads/Code/lumenx/.env)` 中已经填写：

```env
DASHSCOPE_API_KEY=sk-你的_dashscope_key
```

然后**重启后端**。因为后端不会在旧进程里自动重新加载 `.env`。

### 4. FFmpeg 安装失败或不可执行

不要依赖仓库里的 `[bin/ffmpeg](/Users/admin/Downloads/Code/lumenx/bin/ffmpeg)`。当前机器上应直接安装系统 `ffmpeg`：

```bash
HOMEBREW_API_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api \
HOMEBREW_BOTTLE_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles \
brew install ffmpeg
```

- `oss_configured=false`
